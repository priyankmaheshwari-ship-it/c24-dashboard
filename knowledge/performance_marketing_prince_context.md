### SQL queries (GAQL — Google Ads Query Language)

Campaign performance (NB/UAC Buyer + Seller):
```sql
SELECT campaign.id, campaign.name, campaign.status, campaign.advertising_channel_type,
  campaign_budget.amount_micros, metrics.cost_micros, metrics.impressions, metrics.clicks,
  metrics.ctr, metrics.average_cpc
FROM campaign
WHERE segments.date DURING LAST_30_DAYS AND campaign.status != 'REMOVED'
ORDER BY metrics.cost_micros DESC LIMIT 1000
```

Conversion breakdown (per campaign):
```sql
SELECT campaign.id, segments.conversion_action_name, metrics.all_conversions
FROM campaign
WHERE segments.date DURING LAST_30_DAYS AND campaign.status != 'REMOVED'
```

Daily trends:
```sql
SELECT segments.date, metrics.cost_micros, metrics.clicks, metrics.impressions
FROM campaign
WHERE segments.date DURING LAST_30_DAYS AND campaign.status != 'REMOVED'
ORDER BY segments.date
```

Geographic breakdown:
```sql
SELECT segments.geo_target_city, metrics.cost_micros, metrics.clicks, metrics.impressions
FROM geographic_view
WHERE segments.date DURING LAST_30_DAYS
ORDER BY metrics.cost_micros DESC LIMIT 500
```

Keywords (NB only):
```sql
SELECT ad_group_criterion.keyword.text, ad_group_criterion.keyword.match_type,
  ad_group_criterion.quality_info.quality_score, campaign.name, ad_group.name,
  metrics.cost_micros, metrics.clicks, metrics.impressions, metrics.ctr, metrics.average_cpc
FROM keyword_view
WHERE segments.date DURING LAST_30_DAYS AND campaign.status != 'REMOVED'
ORDER BY metrics.cost_micros DESC LIMIT 500
```

Search terms (NB only):
```sql
SELECT search_term_view.search_term, campaign.name, ad_group.name,
  metrics.cost_micros, metrics.clicks, metrics.impressions, metrics.ctr
FROM search_term_view
WHERE segments.date DURING LAST_30_DAYS AND campaign.status != 'REMOVED'
ORDER BY metrics.cost_micros DESC LIMIT 500
```

### ETL & pipeline notes
- **Meta**: Direct browser fetch from Graph API → client-side `fetchInsights()` → 10-min `_fetchCache` Map → `calcMetrics()` → `applyFilter()` → component render
- **Google**: Flask proxy (localhost:8765) → Google Ads API via `google-ads` Python library → 5-min server-side cache → JSON response → React `useGadsLive()` hook → component
- **Inventory**: Google Sheet published CSV → CORS proxy (corsproxy.org fallback) → `parseCSV()` → `normalizeRows()` (URL city extraction + validation) → 24-hour localStorage cache
- **Meta API `until` is EXCLUSIVE** — must add +1 day to include the intended end date
- **Google API date ranges**: GAQL supports `DURING LAST_N_DAYS` or `BETWEEN 'date' AND 'date'` (inclusive)
- **Conversion query pattern**: Two separate queries (perf + conv) merged by campaign_id to avoid row explosion from conversion segmentation
